── S9.5: SHADER ANIMATION SECTION ────────────────────────────────────────────

  TYPE:         Full-screen immersive WebGL/GLSL shader background section
                with overlaid content and CTA — a pure "wow" moment
  BACKGROUND:   Rendered entirely by shader (replaces solid bg colour)

                and creates the most memorable scroll moment on the site
  HEIGHT:       100vh (full viewport height)

  ── SHADER SPECIFICATION ────────────────────────────────────────────────────

    SHADER TYPE:    Fragment shader (GLSL / Three.js / WebGL canvas)
    LIBRARY:        Three.js (r128+) or raw WebGL canvas element
    CANVAS:         Full-bleed, position absolute, z-index 0
                    Renders behind all text/UI content

    VISUAL CONCEPT: "Signal vs Noise" — the metaphor of separating
                    truth from misinformation, visualised as a dynamic
                    particle field where order emerges from chaos.

    EFFECT DESCRIPTION:
      Phase 1 (chaos):
        Thousands of tiny particles (2–3px dots) moving in random
        Brownian motion — simulating the noise and chaos of unverified
        information flooding the web. Colour: cool blue-grays.

      Phase 2 (clarification):
        On scroll enter (Intersection Observer), particles begin to
        self-organise — clustering into structured formations. Warm
        blue particles represent verified truth; red particles (fewer)
        peel off and drift away, representing debunked misinformation.

      Phase 3 (steady state):
        A flowing, calm particle field with subtle wave distortion.
        The overall impression: order from chaos, signal from noise.

    SHADER PARAMETERS:
      uTime:          uniform float — drives animation loop
      uResolution:    uniform vec2  — canvas width/height for aspect ratio
      uMouse:         uniform vec2  — mouse position (0.0–1.0 normalised)
                      Particles subtly repel from cursor position
      uScrollProgress: uniform float — 0.0 to 1.0, drives phase transitions
      uParticleCount: 3000–5000 (depends on device performance)

    COLOUR PALETTE (hardcoded, not CSS variables — physical colour scene):
      Background:     #050508  (near-black with faint blue tint)
      Particle true:  #60A5FA  (soft blue — verified/signal particles)
      Particle false: #F87171  (soft red — misinformation/noise particles)
      Particle neutral:#6B7280 (gray — unclassified/ambient particles)
      Glow centre:    rgba(37,99,235,0.15) — faint blue radial at canvas centre

    PERFORMANCE GUARDS:
      - Detect prefers-reduced-motion → disable animation, show static bg
      - Detect mobile → reduce particle count to 1000, disable mouse tracking
      - Detect low-end GPU (canvas benchmark) → fallback to CSS animation

    FALLBACK (no WebGL):
      CSS animated background with radial pulses and opacity transitions
      Same colour palette, simpler motion

  PART 4: KEY DESIGN DECISIONS INFORMED BY WQF ANALYSIS
────────────────────────────────────────────────────────────────────────────────

1. DARK THEME IS NON-NEGOTIABLE
   WQF uses a deep dark navy (#0D0F1A) throughout. This communicates
   authority, sophistication, and premium quality. Your app should do the same.
   Light theme feels less credible for an AI intelligence tool.

2. SINGLE BOLD ACCENT COLOR
   WQF uses one red accent (#E63946) sparingly for CTAs and brand moments.
   Yours should use blue (#2563EB) as primary — blue communicates trust,
   accuracy, and authority (appropriate for fact-checking).
   Reserve red/amber/green ONLY for verdict colours (False/Misleading/True).

3. LARGE DISPLAY TYPOGRAPHY
   WQF uses oversized headlines (50–70px) with tight letter-spacing (-2 to -3px).
   Match this weight. Headlines should feel confident and declarative.

4. MINIMAL NAVIGATION
   WQF has exactly 5 nav links. Don't add more. Every extra link dilutes CTA.
   Keep: How it works · Features · Agents · API · Contact + "Try free" button.

5. INTERACTIVE PROOF OVER STATIC CLAIMS
   WQF can say "we build great companies" and be believed (established brand).
   You CANNOT — you must DEMONSTRATE. Every claim must be backed by:
   - Live demo users can try immediately
   - Stats with specific numbers (not "thousands of articles")
   - Methodology link for academics + journalists

6. THREE-AUDIENCE STRUCTURE
   WQF organises its pitch around 3 audiences: Founders, Investors, Partners.
   Mirror this for: Journalists, Researchers, Developers.
   Each audience has different needs → different CTAs → different landing pages.

7. FOOTER AS TRUST SIGNAL
   WQF's footer is minimal (just nav + legal). Yours must be more dense because:
   - You're a newer product that needs to signal legitimacy
   - Developers need quick access to API docs and GitHub
   - Researchers need methodology and citation links
   - "Responsible AI policy" is essential — you're making truth claims about news

8. CONTACT FORM PLACEMENT
   WQF embeds the contact form directly on the page (not a separate page).
   Do the same — it removes friction significantly.
   Keep the role/use-case dropdown so you can route responses appropriately.

9. SHADER ANIMATION AS BRAND IDENTITY MOMENT
   WQF uses high-quality photography and SVG icons as its visual language.
   Your app has no real-world photos to draw on — it's a software tool.
   The shader particle animation fills the role that photography plays on WQF:
   it creates an emotional, atmospheric moment that makes the page feel premium.
   Used sparingly (3 places: hero ambient, loading state, dedicated section),
   it becomes a recognisable brand signature without overwhelming the UI.
   Performance budget: shader must not block interaction, pause when off-screen.

10. DISCLAIMER AS TRUST INFRASTRUCTURE (NOT AN AFTERTHOUGHT)
    WQF doesn't need disclaimers — venture advice is obviously human judgment.
    Your app makes automated truth claims about news content — this creates
    significant legal and ethical exposure if users over-rely on the output.
    The disclaimer is not a legal checkbox — it is core product UX:
    - Position it directly below the heatmap (the highest-stakes output)
    - Make it persistent (cannot be dismissed)
    - Include colour legend so users understand highlight semantics
    - Use expandable detail so advanced users can understand model limits
    - Link to Methodology, Terms, and Acceptable Use — building trust loop
    A well-designed disclaimer INCREASES credibility; a missing one destroys it.

11. COMMUNITY AS TRUST MULTIPLIER
    WQF doesn't need a community — it operates with closed, curated networks.
    Your app fights misinformation: collective human verification alongside AI
    is both more accurate and more defensible than either alone.
    - Discussion threads let domain experts (doctors, lawyers, economists)
      flag what the AI missed — catching edge cases the model cannot handle
    - Live sessions create shared accountability: many eyes on one article
    - The community archive becomes a proprietary dataset over time,
      with crowd-verified labels that can be used to retrain the model
    - Design the community as read-only-first: browsing needs no account;
      posting requires free sign-up; moderation uses human + AI layered approach
    - Community verdicts shown side-by-side with AI verdicts — never replacing,
      always supplementing — preserving the product's epistemic honesty
